# Opening pull requests

Open PRs with the `gh` CLI against `SigNoz/terraform-provider-signoz`.

- **Run the checks first.** For a Go change, run the primus `go-fmt` / `go-lint` / `go-deps` / `go-test` gates (see the `go-checks` rule); for docs/examples, the `docs-writer` gates. Don't open a PR on red.
- **Follow the template** (`.github/pull_request_template.md`): put each change under the right heading (Features / Fixes / Chores / Refactors / Tests). Don't add sections the template doesn't have.
- **Keep only the headings that apply.** Delete every heading that has nothing under it, along with its `<Add ...>` placeholder. The body must never contain an empty heading — if only one category applies, the body has exactly that one heading.
- **Keep the description concise and human-readable.** A few plain bullets saying what changed and why, for a reviewer skimming it — not a wall of text, not a restatement of the diff, not generated boilerplate.
- **No AI attribution anywhere in the title or body** — no "Generated by Claude Code" footer, and no "generated by" / "co-authored-by an AI" line of any kind. (A requester-attribution line may be prepended automatically; leave it as-is.)
